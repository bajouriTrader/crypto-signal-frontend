# SignalDesk Backend — تاریخچه‌ی نسخه‌ها (CHANGELOG)

هر پوشه‌ی `V.X` یک **snapshot کامل** از فایل‌های تغییریافته‌ی بک‌اند است — یعنی
برای هر نسخه، همه‌ی فایل‌هایی که باید روی HuggingFace Space باشند داخل همون
پوشه‌ست (نه فقط diff). برای برگشت به نسخه‌ی قبلی، کافیه فایل‌های همون پوشه
رو کامل جایگزین کنی — نیازی به فهمیدن این‌که دقیقاً چی عوض شده نیست.

بعد از دیپلوی هر نسخه، با درخواست `GET /version` می‌تونی مطمئن بشی دقیقاً
کدوم نسخه روی سرور در حال اجراست (هر نسخه یک `APP_VERSION` مشخص داره).

---

## V.0 — نسخه‌ی پایه (فعلاً روی سرور شماست)

شامل ۳ فیکس باگی که قبلاً با هم انجام دادیم:
1. سکوت خطای PATCH به Supabase (الان لاگ می‌شه + retry + reconcile خودکار)
2. توقف چک تایم‌اوت وقتی گرفتن کندل از توبیت شکست می‌خورد (الان fallback داره)
3. ستون گمشده‌ی `realized_pnl_percent` در جدول Supabase (این یکی SQL بود، نه کد — قبلاً روی Supabase اجرا شده و مربوط به هیچ‌کدوم از این فایل‌ها نیست)

فایل‌های این نسخه:
- `app.py`
- `demo_trade.py`
- `indicators.py`
- `supabase_client.py`

## V.1 — فاز ۱ نقشه‌راه بهبود وین‌ریت

روی پایه‌ی V.0، این‌ها اضافه شد:
- غیرفعال‌سازی حالت **strict** (وین‌ریت واقعی‌اش ۵.۶٪ روی ۱۸ معامله بود)
- حداقل **confluence_score = 55** برای این‌که سیگنال «قابل‌ورود» حساب بشه (قبلاً فقط برای نمایش/مرتب‌سازی بود)
- فیلتر **نماد/جهت** بر اساس وین‌ریت واقعی گذشته (از `/demo-trade/stats`) — نمادی با ≥۱۰ معامله و وین‌ریت <۴۰٪، یا جهتی با ≥۳۰ معامله و وین‌ریت <۳۵٪، موقتاً بلاک می‌شه
- Endpoint جدید `GET /signal-filters/status` برای دیدن وضعیت فعلی این فیلترها در پنل ادمین

فایل‌های این نسخه (همه باید با هم جایگزین/اضافه بشن):

**بک‌اند** (کنار هم در ریشه‌ی پروژه‌ی HuggingFace):
- `app.py` (تغییر یافته)
- `indicators.py` (تغییر یافته)
- `signal_filters.py` (**جدید**)
- `demo_trade.py` (بدون تغییر نسبت به V.0 — فقط برای کامل بودن snapshot اینجاست)
- `supabase_client.py` (بدون تغییر نسبت به V.0 — فقط برای کامل بودن snapshot اینجاست)

**فرانت‌اند** (کنار هم در `src/` پروژه‌ی GitHub Pages):
- `App.jsx` (تغییر یافته — نمایش شماره‌ی نسخه‌ی فرانت + بک‌اند بالای صفحه، کنار لوگوی SignalDesk. بک‌اندش از `GET /version` گرفته می‌شه)
- `index.css` (تغییر یافته — استایل کوچیک برای نشان نسخه)

### نحوه‌ی دیپلوی V.1
همه‌ی فایل‌های بک‌اند رو در ریشه‌ی پروژه‌ی بک‌اند، و فایل‌های فرانت‌اند رو در `src/` پروژه‌ی فرانت جایگزین/اضافه کن و هرکدوم رو جدا دیپلوی بزن (بک‌اند روی HuggingFace، فرانت روی GitHub Pages).

### نحوه‌ی برگشت از V.1 به V.0 (در صورت نیاز)
فقط فایل‌های پوشه‌ی `V.0/` رو دوباره جایگزین کن (و `signal_filters.py` رو از HuggingFace حذف کن، چون در V.0 وجود نداره — البته حذف نکردنش هم مشکلی ایجاد نمی‌کنه چون هیچ فایلی دیگه importش نمی‌کنه).
