# SignalDesk Backend — تاریخچه‌ی نسخه‌ها (CHANGELOG)

هر پوشه‌ی `V.X` یک **snapshot کامل** از فایل‌های تغییریافته‌ی بک‌اند است — یعنی
برای هر نسخه، همه‌ی فایل‌هایی که باید روی HuggingFace Space باشند داخل همون
پوشه‌ست (نه فقط diff). برای برگشت به نسخه‌ی قبلی، کافیه فایل‌های همون پوشه
رو کامل جایگزین کنی — نیازی به فهمیدن این‌که دقیقاً چی عوض شده نیست.

بعد از دیپلوی هر نسخه، با درخواست `GET /version` می‌تونی مطمئن بشی دقیقاً
کدوم نسخه روی سرور در حال اجراست (هر نسخه یک `APP_VERSION` مشخص داره).

## V.0 — نسخه‌ی پایه (فعلاً روی سرور شماست)

شامل ۳ فیکس باگی که قبلاً با هم انجام دادیم:
1. سکوت خطای PATCH به Supabase (الان لاگ می‌شه + retry + reconcile خودکار)
2. توقف چک تایم‌اوت وقتی گرفتن کندل از توبیت شکست می‌خورد (الان fallback داره)
3. ستون گمشده‌ی `realized_pnl_percent` در جدول Supabase (این یکی SQL بود، نه کد — قبلاً روی Supabase اجرا شده و مربوط به هیچ‌کدوم از این فایل‌ها نیست)

فایل‌های این نسخه:
- `app.py`
- `demo_trade.py`
- `indicators.py`
- `supabase_client.py`

## V.1 — فاز ۱ نقشه‌راه بهبود وین‌ریت

روی پایه‌ی V.0، این‌ها اضافه شد:
- غیرفعال‌سازی حالت **strict** (وین‌ریت واقعی‌اش ۵.۶٪ روی ۱۸ معامله بود)
- حداقل **confluence_score = 55** برای این‌که سیگنال «قابل‌ورود» حساب بشه (قبلاً فقط برای نمایش/مرتب‌سازی بود)
- فیلتر **نماد/جهت** بر اساس وین‌ریت واقعی گذشته (از `/demo-trade/stats`) — نمادی با ≥۱۰ معامله و وین‌ریت <۴۰٪، یا جهتی با ≥۳۰ معامله و وین‌ریت <۳۵٪، موقتاً بلاک می‌شه
- Endpoint جدید `GET /signal-filters/status` برای دیدن وضعیت فعلی این فیلترها در پنل ادمین

فایل‌های این نسخه (همه باید با هم جایگزین/اضافه بشن):

**بک‌اند** (کنار هم در ریشه‌ی پروژه‌ی HuggingFace):

- `indicators.py` (تغییر یافته)
- `signal_filters.py` (**جدید**)
- `demo_trade.py` (بدون تغییر نسبت به V.0 — فقط برای کامل بودن snapshot اینجاست)
- `supabase_client.py` (بدون تغییر نسبت به V.0 — فقط برای کامل بودن snapshot اینجاست)

**فرانت‌اند** (کنار هم در `src/` پروژه‌ی GitHub Pages):

## نسخه‌های بعدی

هر فاز جدید از نقشه‌راه (فاز ۲ به بعد) یک پوشه‌ی `V.2`, `V.3`, ... جدید با همین ساختار (snapshot کامل + توضیح تغییرات این‌جا) خواهد بود. اصلاحات کوچک روی یک نسخه‌ی موجود (نه فاز کاملاً جدید) با شماره‌ی اعشاری همون نسخه بالا می‌رن (مثل V.1 → V.1.1 → V.1.2، یا V.2 → V.2.1).

## V.1.1 — اصلاح بر مبنای بازبینی دقیق تصمیم‌های V.1

بعد از دیپلوی V.1، کاربر ۳ نگرانی درباره‌ی خودِ طراحی فاز ۱ مطرح کرد که دو موردش باعث اصلاح واقعی کد شد:

### ۱. آیا گیت `MIN_CONFLUENCE_SCORE = 55` واقعاً چیزی رو فیلتر می‌کنه؟
**بررسی شد و پاسخ: بله.** برخلاف نگرانی اولیه (که فرض می‌کرد امتیاز فقط ۴ مقدار ثابت داره)، فرمول واقعی در `indicators.py` پیوسته است:
```
base_score = (25 برای relaxed) + (15 اگه alignment تمیز) + market_score_adj (بین ۱۸- تا ۱۲+) + session_bonus (۰ یا ۵) + (m30_votes + m15_votes) × 4
final_score = base_score + (۱۵ اگه تریگر ۵M فعال)
```
بازه‌ی واقعی برای یک سیگنال ساده‌گیرِ تریگرخورده حدود ۳۰ تا ۱۰۰ است، پس آستانه‌ی ۵۵ واقعاً بخشی از سیگنال‌ها (مخصوصاً وقتی BTC مخالف جهته و رأی‌های RSI/MACD/ADX/حجم کمن) رو فیلتر می‌کنه.
برای این‌که این ادعا قابل‌راستی‌آزمایی مستقیم باشه (نه فقط از روی کد)، ستون **«امتیاز Confluence»** به خروجی مارک‌داون و اکسل پنل ادمین اضافه شد (`AdminPanel.jsx`) — از این به بعد می‌شه توزیع واقعی امتیازها رو مستقیم از گزارش دید.

### ۲. حجم نمونه‌ی فیلتر نماد/جهت کوچک بود + یک باگ طراحی واقعی
نگرانی درست بود: با ۱۰ معامله، حتی یک استراتژی با ادج واقعی ۵۰٪ می‌تونه شانسی زیر ۴۰٪ بیفته. **همچنین یک نقص طراحی واقعی هم پیدا شد:** چون نماد بلاک‌شده دیگه معامله‌ی جدید نمی‌گیره، آمارش هم هیچ‌وقت بروز نمی‌شه — یعنی بلاک می‌تونست عملاً **دائمی** بشه.

اصلاحات:
- `MIN_SAMPLE_SIZE_SYMBOL`: ۱۰ → **۲۰**
- `MIN_SAMPLE_SIZE_DIRECTION`: ۳۰ → **۵۰**
- مکانیزم جدید **"دوره‌ی آزمایشی" (probation)**: هر نماد/جهت بلاک‌شده، حداکثر یک‌بار در روز و با احتمال ۱۵٪، یک شانس معامله‌ی آزمایشی می‌گیره تا آمارش بتونه به‌مرور خودش رو تصحیح کنه — بلاک هیچ‌وقت قطعی نیست.

### ۳. شفاف‌سازی وضعیت strict
نگرانی درست بود که تعلیق strict باید جایی صریحاً «موقت» ثبت بشه، نه «رد قطعی». اضافه شد:
- فیلد جدید `STRICT_MODE_STATUS = "paused_pending_redesign"` (به‌جای فقط یک بولین)
- در `GET /signal-filters/status` هم این فیلد و هم تنظیمات probation نمایش داده می‌شه.

فایل‌های این نسخه:

**بک‌اند:**
- `app.py` (تغییر یافته — نسخه + endpoint status)
- `signal_filters.py` (تغییر یافته — آستانه‌های نمونه + مکانیزم probation)
- `indicators.py`, `demo_trade.py`, `supabase_client.py` (بدون تغییر نسبت به V.1 — فقط برای کامل بودن snapshot)

**فرانت‌اند:**
- `App.jsx` (تغییر یافته — فقط نسخه به V.1.1)
- `AdminPanel.jsx` (**جدید در این پوشه، ولی فایل خودِ پروژه از قبل وجود داشت** — ستون امتیاز Confluence به هر دو خروجی مارک‌داون و اکسل اضافه شد)
- `index.css` (بدون تغییر نسبت به V.1 — فقط برای کامل بودن snapshot)
